# 진(jin) RPG 구상 v3

작성일: 2026-06-11

## 이름

게임 이름은 `진(jin) RPG`로 한다.

## 핵심 방향

진 RPG는 포켓몬스터나 바람의 나라처럼 필드를 돌아다니며 몬스터를 만나고 성장하는 2D 필드 액션 RPG다.

전투는 별도 전투 화면으로 진입하지 않는다. 플레이어, 몬스터, 펫이 같은 맵 위에서 이동하고 공격하며, 각 객체 머리 위에 HP bar가 표시된다.

핵심 재미는 몬스터 사냥 중 낮은 확률로 `죽은 코어`가 드랍되고, 이 코어를 마을 NPC에게 부활시켜 최대 3마리까지 펫으로 데리고 다니며 함께 싸우는 것이다.

## 조작과 전투 감각

- 왼쪽 하단 조이스틱으로 이동한다.
- 오른쪽 하단에 일반 공격 버튼과 스킬 버튼을 둔다.
- 공격 버튼을 누르면 바라보는 방향 또는 가까운 타겟을 공격한다.
- 몬스터는 맵을 돌아다니다가 플레이어를 감지하면 접근해 공격한다.
- 플레이어가 몬스터를 공격하거나 피격되면 동행 펫도 같은 전투에 참여한다.
- 펫은 플레이어 타겟을 우선 공격한다.
- 전투 상태에서도 이동, 회피, 거리 조절이 가능해야 한다.

## 코어와 펫

몬스터를 처치하면 경험치, 골드, 아이템과 함께 낮은 확률로 죽은 코어가 드랍될 수 있다.

초반 몬스터 코어는 비교적 체감 가능하게 드랍되지만, 중반 이후 희귀 몬스터와 보스 코어는 매우 낮은 확률로 설계한다. 천장 보상이나 확정 보상 루프보다는 희귀 드랍 자체의 기대감이 핵심이다.

죽은 코어는 바로 사용할 수 없고, 마을의 코어 관련 NPC에게 부활시켜야 한다.

부활한 코어는 펫으로 장착할 수 있으며, 최대 3마리까지 동행한다.

## 코어 속성

코어에는 몬스터 이름 앞에 붙는 속성이 있다. 속성은 몬스터의 기본 능력치에 성장 보정을 주는 방식으로 처리한다.

| 속성 | 성장 보정 방향 | 전투 체감 |
|---|---|---|
| 힘센 | 힘 중심 성장 | HP와 공격력이 높다 |
| 날쌘 | 민첩 중심 성장 | 공격속도, 이동, 회피 체감이 좋다 |
| 정확한 | 정확 중심 성장 | 명중과 크리티컬 안정성이 좋다 |
| 운좋은 | 행운 중심 성장 | 회피, 크리티컬, 드랍 보조 체감이 있다 |
| 속성강한 | 속성 중심 성장 | 속성 공격과 속성 방어가 좋다 |

몬스터별 스탯은 기본값 중심으로 관리하고, 속성별 보정은 별도 테이블 또는 데이터로 처리한다.

## 성장 구조

플레이어는 몬스터를 처치해 경험치와 골드를 얻고 레벨업한다.

성장 요소는 다음을 기본으로 한다.

- 레벨
- 경험치
- 골드
- HP
- MP
- 힘
- 민첩
- 정확
- 행운
- 속성
- 장비 스탯
- 스킬

펫도 전투 경험치 일부를 얻어 성장한다. 펫 성장은 플레이어보다 느리게 하되, 오래 데리고 다닌 펫이 강해지는 체감은 있어야 한다.

## 마을과 필드

마을은 정비와 성장의 중심이다.

마을 기능:

- 코어 부활 NPC
- 잡화상
- 무기상
- 장비상
- 창고
- 펫 관리

필드는 사냥과 드랍의 중심이다.

초기 지역:

- 초보자의 마을
- 푸른 초원

초기 몬스터:

- 슬라임
- 잎버섯
- 들박쥐
- 어린 늑대
- 왕슬라임

## 아이템과 장비

아이템은 전투 유지와 성장 보조 역할을 한다.

초기 아이템:

- 작은 HP 물약
- 작은 MP 물약
- 마을 귀환서
- 펫 회복제

장비는 플레이어 전투 스타일을 바꾸는 방향으로 설계한다.

장비 종류:

- 무기
- 방어구
- 장신구

무기는 사거리, 공격속도, 공격력 체감이 달라야 한다.

## 스킬

초기에는 버튼 수를 과하게 늘리지 않는다.

1차 구현 기준:

- 일반 공격
- 베기
- 돌진타

스킬은 MP와 쿨타임을 사용한다.

## MVP 범위

1차 목표는 전체 RPG 완성이 아니라, 첫 펫 획득까지의 핵심 루프를 완성하는 것이다.

MVP 완료 기준:

- 홈에서 진 RPG 진입
- 이름 입력 후 플레이어 생성 또는 불러오기
- 초보자의 마을과 푸른 초원 이동
- 모바일 조이스틱 이동
- 카메라 follow
- 몬스터 roaming, aggro, 추적, 복귀, respawn
- 맵 위 실시간 공격과 피격
- 플레이어와 몬스터 HP bar
- 몬스터 처치 보상
- 죽은 코어 드랍
- 코어 부활 NPC
- 펫 1마리 동행
- Supabase 저장과 불러오기

## 초반 10분 플레이 동선

초반 10분 안에 게임의 핵심 재미가 보여야 한다.

| 시간 | 목표 |
|---|---|
| 1분 | 이름 입력, 마을 시작 |
| 2분 | 초원 이동, 슬라임과 첫 전투 |
| 4분 | 레벨 2 달성 |
| 6분 | HP 물약 필요성 체감 |
| 8분 | 첫 죽은 코어 획득 가능성 체감 |
| 10분 | 코어 부활 NPC 사용, 첫 펫 동행 |

## 구현 구조 방향

기존 `js/app.js`에 모든 RPG 코드를 붙이지 않는다. 진 RPG는 별도 폴더로 분리해서 키운다.

권장 구조:

```text
js/
  games/
    jinrpg/
      jinrpg.js
      data.js
      engine.js
      input.js
      combat.js
      pets.js
      save.js
      ui.js
css/
  jinrpg.css
sql/
  supabase_jinrpg.sql
```

기존 앱에는 홈 카드, 메뉴 버튼, 진입점만 최소로 연결한다.

## Supabase 저장 방향

정적 데이터는 JS 파일로 관리하고, 플레이어 진행 데이터만 Supabase에 저장한다.

필요 테이블 초안:

- `jinrpg_players`
- `jinrpg_pet_cores`
- `jinrpg_pet_slots`
- `jinrpg_inventory`
- `jinrpg_equipment`
- `jinrpg_drop_logs`
- `jinrpg_kill_counts`

Supabase SQL은 별도 파일로 작성해서 사용자가 SQL Editor에서 실행한다.
